## 실시간 웹툰 이미지 스크롤 번역 앱 개발

| 구분 | 내용 |
| :--- | :--- |

## 배경 및 문제 정의
**문제점: 웹 이미지 번역의 한계**
외국 웹툰이나 해외 웹사이트를 이용할 때, 크롬 웹 번역 기능은 일반 텍스트만 지원할 뿐, 이미지 내 포함된 텍스트는 번역해주지 않습니다.

**기존 앱 조사 결론**
Google Play 스토어의 기존 무료 이미지 번역 앱들은 대부분 다음과 같은 제한적인 방식으로 작동했습니다.

갤러리 캡처 방식: 이미지를 캡처한 후 갤러리에서 선택해야 번역 가능.

화면 고정 방식: 화면을 캡처하여 번역하는 동안 화면 조작(스크롤 등)이 불가능하여 웹페이지를 실시간으로 탐색할 수 없음.

**→ 결론: 웹페이지에서 실시간으로 스크롤하며 이미지를 번역하여 볼 수 있는 애플리케이션을 만들자!**

## 💡 STAR WORDs 주요 기능 및 구현 설계
### 독창적인 UI/UX (우주 테마)
| 요소 | 디자인/기능 | 구현 방식 |
| :--- | :--- | :--- |
| **메인 기능 버튼** | 화면 위에 떠 있는 플로팅 버튼 형태로 제공. | **Foreground Service**와 **WindowManager**를 활용한 오버레이(Overlay) 구현. |
| **버튼 애니메이션** | 중앙 버튼 클릭 시 서브 버튼 3개가 태양계처럼 등장. | **ObjectAnimator**를 이용해 애니메이션 구현. |
| **스마트 버튼 배치** | 오버레이 버튼이 화면 가장자리로 이동 시 $1/3$ 정도 자동 숨김 및, 좌우 영역에 따라 서브 버튼이 반대 방향으로 나오도록 구현. | 드래그 **Touch Event** 처리 및 위치 인식 로직 구현. |
| **언어 선택** | 우주선 형태의 버튼, Popup Menu 활용. | **SharedPreferences**를 통해 서비스와 메인 화면 간 언어 정보 연동 관리. |
| **테마/인트로** | 메인 화면 배경(우주/은하), 스타워즈 패러디 영상을 앱 시작 전 띄우기. | 디자인 이미지 및 영상 콘텐츠 제작. |

### 기술적 구현 방식
| 기능 | 주요 구현 방식 |
| :--- | :--- |
| **오버레이 뷰** | **Foreground Service**에서 **WindowManager**를 활용해 TextView 및 버튼을 다른 앱 위에 띄움. (Android 8.1 이상 채널 지정 및 알림 생성 포함) |
| **번역 시작 동작** | 번역 시작 버튼 클릭 시 **화면 스캔 → 번역 → 텍스트 띄우기** 동작 순차 수행. |
| **정보 연동** | 언어 선택 및 텍스트 테마 정보를 **SharedPreferences**를 통해 메인 Activity와 서비스 간에 공유 및 관리. |

### 번역 api 비교 및 선정 (성능 중시)
| API | 번역 속도 | 번역 성능 | 선정 이유 |
| :--- | :--- | :--- | :--- |
| **ML Kit** | **가장 빠름** | 가장 낮음 | **성능(정확도)**이 중요하다고 판단하여 $\text{ML Kit}$로 결정 (Google의 머신러닝 $\text{SDK}$). |
| **Google Trans** | 중간 | 중간 | |
| **Deepl** | 가장 느림 | **가장 우수** | |

## 기대 효과
**STAR WORDs**는 이전에 Chrome에서 번역할 수 없었던 웹 이미지 텍스트를 실시간으로 번역합니다.
화면을 캡처하는 방식으로 작동되므로 웹툰의 말풍선뿐만 아니라 Youtube와 같은 영상물의 자막 등 화면에 나타난 모든 텍스트를 번역하는 데 이용될 수 있으며, 사용자들의 웹 탐색 범위를 넓혀줄 것으로 기대됩니다.
